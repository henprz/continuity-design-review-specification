# Changelog

All notable changes to the Continuity Design Review Specification (CDRS) will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project follows the principles of Semantic Versioning for specification releases.

---

## [1.0.0] - 2026-07-19

### Added

- Initial public release of the Continuity Design Review Specification (CDRS).
- Defined the purpose, scope, and design goals of the specification.
- Introduced Quick Review and Guided Review modes.
- Established the review process and report structure.
- Defined the confidence model for inferred information.
- Introduced qualitative evaluation levels.
- Published under the Creative Commons Attribution 4.0 International (CC BY 4.0) license.

## Versioning Policy

CDRS uses Semantic Versioning adapted for specifications.

### Major (`X.0.0`)

Breaking changes that modify the specification in a way that may require existing implementations to change.

Examples:

- New mandatory review steps.
- Changes to report structure.
- Changes to evaluation dimensions.
- Removal of existing concepts.

### Minor (`1.X.0`)

Backward-compatible additions or clarifications.

Examples:

- New optional sections.
- Additional examples.
- Expanded terminology.
- New guidance or recommendations.

### Patch (`1.0.X`)

Editorial improvements that do not change the meaning of the specification.

Examples:

- Grammar corrections.
- Formatting improvements.
- Typographical fixes.
- Clarifications without changing normative behavior.
