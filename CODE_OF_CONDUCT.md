# Code of Conduct

## Purpose

The Continuity Design Review Specification (CDRS) is intended to foster thoughtful discussion about product evaluation, design, and the evolution of the Continuity Design framework.

This Code of Conduct exists to ensure that discussions remain constructive, respectful, and focused on improving the specification.

---

## Expected Behavior

Participants are expected to:

- Treat others with respect and professionalism.
- Discuss ideas rather than individuals.
- Support arguments with reasoning and evidence whenever possible.
- Welcome constructive criticism.
- Accept that disagreement is a natural part of refining a specification.
- Help maintain a collaborative environment.

---

## Unacceptable Behavior

The following behaviors are not acceptable:

- Personal attacks or harassment.
- Discrimination of any kind.
- Offensive or abusive language.
- Trolling or intentionally disruptive behavior.
- Repeatedly derailing technical discussions.
- Misrepresentation of another contributor's position.

---

## Decision Making

CDRS is governed under the Benevolent Dictator For Life (BDFL) model.

Technical discussions and proposals are encouraged. Final decisions regarding the evolution of the specification are made by the BDFL to preserve the conceptual integrity and long-term vision of the project.

---

## Reporting Issues

If you encounter behavior that violates this Code of Conduct, please open a private communication with the project maintainer.

Reports will be handled respectfully and, whenever possible, confidentially.

---

## Scope

This Code of Conduct applies to:

- GitHub Issues
- Pull Requests
- Discussions
- Documentation reviews
- Any official communication channels related to the project

---

## Our Goal

The primary goal of this project is to build a high-quality, implementation-independent specification that benefits the broader product community.

Every discussion should contribute toward that objective.
